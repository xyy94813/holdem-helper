# 产品规格文档：德州扑克 PWA 胜率计算器

## 1. 产品概述

该应用是一款面向德州扑克玩家的 Progressive Web App（PWA），旨在帮助用户快速评估当前手牌在已翻开公共牌情况下的胜率。用户只需输入自己的两张手牌及已揭晓的公共牌，应用即可实时计算该手牌在当前局势中的胜率。

## 2. 核心目标

- 提供德州扑克手牌胜率评估
- 支持基于用户当前手牌和已翻开的公共牌计算胜率
- 作为 PWA 实现，支持离线访问、安装到主屏幕和移动体验

## 3. 目标用户

- 休闲德州扑克爱好者
- 想要提高手牌决策准确性的玩家
- 进行策略练习、模拟牌局或复盘分析的用户

## 4. 关键功能

### 4.1 手牌与公共牌输入
- 手牌输入采用“先选花色，再输入牌号”的组合方式，便于快速选择。
- 公共牌输入支持 0 到 5 张已翻开的公共牌（Flop / Turn / River），其中 Flop 三张牌单独一行显示，Turn / River 另排显示。
- 牌号支持文本输入，格式为 `2-9、T、J、Q、K、A`；花色通过 `h/d/c/s` 选择。
- 输入 UI 应避免重复牌，并实时校验非法牌面
- 提供“清空”与“重置”按钮，方便用户快速重算

### 4.2 胜率计算
- 计算当前手牌在给定公共牌情况下的胜率
- 返回主胜率指标：胜率、平局概率、输牌概率
- 默认计算方式为前端本地计算，采用轻量枚举或 Monte Carlo 模拟
- 仅针对单手牌与随机对手进行计算，不包含用户定义对手牌范围
- 支持实时更新结果，公共牌或手牌变更即可自动重新计算

### 4.3 结果展示
- 清晰展示胜率数值、平局概率和失败概率
- 提供可视化图表（如条形图或饼图）增强理解
- 显示当前牌型提示和可能的最佳组合类型
- 展示当前胜率高/低的简要说明，帮助用户理解当前手牌强度

### 4.4 PWA 能力
- 支持 Web App Manifest 和“添加到主屏幕”功能
- 支持离线缓存关键页面、静态资源和计算模块
- 离线状态下仍可输入牌面并进行本地胜率计算
- 响应式布局，适配手机、平板与桌面屏幕

## 5. MVP 范围

### 必须实现
- 手牌与公共牌输入界面，包括牌面选择、输入校验与错误提示
- 胜率计算引擎，基于当前手牌和已翻牌的本地计算
- 结果展示与实时更新，包含数值和简单可视化
- PWA 安装提示与基础离线缓存，离线时仍可输入和计算

### 可选/后续迭代
- 手牌建议与策略提示
- 多玩家局势模拟
- 历史记录与分析报告
- 牌型概率热力图

## 6. 用户流程

1. 用户打开 PWA
2. 进入“胜率计算器”页面
3. 选择或输入两张手牌
4. 选择或输入已翻开的公共牌
5. 系统自动计算并展示当前胜率

## 7. 成功指标

- 计算结果准确性：胜率计算符合已知牌局概率模型
- 交互效率：用户三步内完成输入和查看结果
- PWA 可安装率：支持“添加到主屏幕”并正常运行
- 响应速度：胜率结果在 1 秒内返回

## 8. 关键技术要求

- 前端框架：React / Vue / Svelte 等可选
- PWA：manifest、service worker、离线缓存策略
- 计算模块：纯前端胜率计算或调用轻量本地库，支持 0~5 张公共牌的实时计算
- 数据验证：确保输入牌面有效、格式正确、无重复
- 性能要求：计算结果在 1 秒内返回，避免浏览器卡顿
- 用户体验：提供可视化输入组件、错误提示和结果解释

## 9. 设计原则

- 简洁：界面直观、步骤少、输入清晰
- 准确：胜率计算结果可信
- 快速：尽量避免长时间等待
- 便捷：支持移动设备、安装与离线使用

## 10. 未来扩展方向

- 加入“模拟对手范围”功能
- 支持不同牌局人数和下注阶段分析
- 提供博弈策略建议和牌型解释
- 支持多语言与主题模式

## 11. 前端工程师问题答复

### 业务范围与边界
- 本产品聚焦于单手牌胜率计算，目标是根据用户当前两张手牌和已揭示公共牌，计算该手牌对随机对手的胜率。
- 当前版本不包含对手手牌范围建模或多玩家局势模拟，这些作为后续扩展。
- 成功指标仅针对胜率计算准确性、交互效率、PWA 安装与离线能力。

### 输入与校验
- 推荐采用扑克牌选择器组件，用户可以点击牌面选择，也可通过文本输入快速录入。
- 输入组件应实时校验牌面合法性，并在发现重复牌或非法输入时立即提示错误。
- 输入交互应包含“清空”和“重置”功能，避免用户操作复杂。

### 胜率计算策略
- 初期采用前端本地计算方案，基于轻量枚举或 Monte Carlo 模拟实现胜率计算。
- 计算应覆盖 0~5 张公共牌，重点保证结果在常见牌局下具有可信度。
- 若前端性能受限，可考虑集成轻量计算库或使用 Web Worker 异步执行计算。

### 结果展示与用户理解
- 结果页面应展示胜率、平局概率、败率，以及当前牌型提示。
- 增加简单解释文本，例如“当前手牌在这轮中稳定性较高”或“目前需要关注对手可能的高牌组合”，提升用户理解。
- 可视化形式宜采用条形图、饼图或状态色块，突出结果变化。

### PWA 离线能力
- 重点支持静态资源与计算模块离线缓存，保证无网络时仍可输入牌面并进行本地计算。
- 离线模式下的用户体验应保持可安装、可打开、可使用计算结果。
- 网络恢复后可同步最新版本与缓存策略更新。
